# 个人作品集网站 PRD

版本：v1.0
日期：2026-05-28
项目名称：个人作品集网站
项目类型：个人品牌 / 作品展示 / 简历补充型网站

---

## 1. 项目背景

用户希望搭建一个属于自己的个人作品集网站，用于展示个人介绍、项目经历、职业能力、成长轨迹与联系方式。网站整体风格参考极简、开发者感、内容驱动型个人主页：页面结构清晰，视觉轻量，强调真实经历、项目成果与长期积累，而不是过度包装。

该网站既要能作为个人品牌主页，也要能在求职、自由职业、项目合作、社交媒体介绍、简历链接等场景中使用。

---

## 2. 产品目标

### 2.1 核心目标

1. 建立一个清晰、可信、长期可维护的个人展示入口。
2. 让访问者在 30 秒内理解“我是谁、我会什么、我做过什么、如何联系我”。
3. 通过重点项目、时间线、博客/文章等内容，体现个人能力与成长过程。
4. 支持后续持续更新，不依赖复杂后台即可维护内容。

### 2.2 非目标

1. 第一版不做复杂 CMS 后台。
2. 第一版不做登录、评论、用户系统。
3. 第一版不追求强商业化转化页风格。
4. 第一版不照搬参考网站视觉，只提取其结构、节奏与极简风格。

## 3. 目标用户

### 3.1 主要访问者

1. 招聘方 / HR / 面试官
   关注候选人的能力、项目质量、技术栈、履历与联系方式。

2. 潜在合作方 / 客户
   关注过往案例、个人可信度、交付能力与沟通入口。

3. 同行业朋友 / 开发者 / 设计师
   关注项目细节、技术文章、开源链接、个人观点。

4. 普通访客
   通过社交媒体、搜索或名片链接进入，快速了解个人背景。

## 4. 用户需求

### 4.1 招聘方需求

* 快速看到姓名、定位、技能方向。
* 查看代表项目与项目成果。
* 获取简历、GitHub、LinkedIn、邮箱等链接。
* 判断候选人的持续学习能力与真实经历。

### 4.2 合作方需求

* 了解服务能力和过往案例。
* 确认是否有相关行业或技术经验。
* 快速发起联系。

### 4.3 站主本人需求

* 能方便更新项目、经历、文章与联系方式。
* 网站加载快、稳定、易部署。
* 视觉简洁，突出内容。
* 适合长期积累，不因短期审美变化迅速过时。

## 7. 页面功能需求

## 7.1 首页

### 页面目标

让访问者快速理解个人身份、能力方向、代表作品和联系方式。

### 页面模块

#### 7.1.1 顶部导航

导航项建议：

* Home
* Projects
* Blog
* Timeline
* CV
* Contact

桌面端：横向导航。
移动端：折叠菜单或简化横向滚动导航。

#### 7.1.2 Hero 个人介绍区

展示内容：

* 头像或个人标识
* 姓名
* 一句话身份定位
* 简短自我介绍
* 主要 CTA

示例结构：

姓名：XXX
定位：Product Designer / Frontend Developer / AI Builder
简介：我关注产品体验、前端开发与 AI 工具，喜欢把想法做成真实可用的产品。
CTA：查看项目 / 下载简历 / 联系我

#### 7.1.3 社交与联系链接

展示常用链接：

* Email
* GitHub
* LinkedIn
* X / Twitter
* Instagram / 小红书 / 即刻，可选
* Blog
* Resume
* WeChat，可通过弹窗或二维码展示

要求：

* 链接应有图标或清晰文本。
* 邮箱支持点击复制或 mailto。
* 外链新标签页打开。
* 移动端点击区域足够大。

#### 7.1.4 重点项目区

展示 3–6 个最重要项目。

每个项目卡片包含：

* 项目名称
* 项目封面图或图标
* 一句话简介
* 个人角色
* 技术栈 / 工具
* 项目成果
* 访问链接
* 源码链接，可选
* 案例详情链接，可选

项目排序原则：

1. 最能代表当前能力的项目优先。
2. 有实际用户、数据、上线成果的项目优先。
3. 与目标职业方向匹配的项目优先。

#### 7.1.5 时间线区

用于展示成长经历、工作经历、项目节点、学习节点、获奖或重要里程碑。

每条时间线包含：

* 时间
* 标题
* 副标题
* 描述
* 标签，可选
* 相关链接，可选

时间线默认倒序排列，即最新经历在最上方。

#### 7.1.6 联系区

页面底部提供清晰联系方式。

内容包括：

* 一句邀请语
* 邮箱
* 社交链接
* 可选：微信二维码
* 可选：所在地 / 可合作方式

示例：

“正在寻找有趣的产品、前端或 AI 项目合作，欢迎联系我。”

## 8. 视觉设计要求

## 8.1 整体风格

参考方向：

* 极简
* 干净
* 低饱和度
* 内容优先
* 开发者气质
* 轻微动态背景或粒子效果
* 卡片式项目展示
* 时间线叙事

## 8.2 色彩

推荐方案：

* 背景色：深色或浅色均可，建议支持暗色模式。
* 主色：选择一种高识别度强调色，例如蓝、紫、绿、橙。
* 文本色：保证对比度清晰。
* 辅助色：用于标签、边框、分割线、状态提示。

第一版建议采用：

* 默认暗色主题
* 支持浅色主题切换
* 强调色用于链接、按钮、时间线节点和项目标签

## 8.3 字体

中文字体建议：

* 系统默认字体优先
* PingFang SC
* Microsoft YaHei
* Noto Sans SC

英文字体建议：

* Inter
* IBM Plex Sans
* JetBrains Mono，用于代码和技术标签

## 8.4 布局

桌面端：

* 内容最大宽度建议 960–1120px。
* 首页 Hero 可采用左文右图，或居中布局。
* 项目区采用 2 列或 3 列卡片。
* 时间线采用单列纵向结构。

移动端：

* 单列布局。
* 卡片宽度 100%。
* 导航简化。
* 社交链接可换行展示。

## 8.5 动效

动效原则：

* 克制
* 不干扰阅读
* 加强层次感

可选动效：

* 页面进入淡入
* 项目卡片 hover 上浮
* 链接 hover 下划线或变色
* 时间线节点渐入
* 背景粒子缓慢运动

禁用：

* 大量炫技动画
* 影响加载性能的 3D 效果
* 闪烁频率过高的动效

## 11. 技术建议

### 11.1 技术栈方案 A：轻量静态站

适合：个人维护、内容更新频率中等、追求速度。

建议：

* Astro / Next.js / Nuxt
* Markdown / MDX 管理内容
* Tailwind CSS
* Vercel / Netlify 部署
* GitHub 管理源码

### 11.2 技术栈方案 B：博客与作品集兼顾

适合：后续会持续写文章。

建议：

* Next.js
* MDX
* Contentlayer 或类似内容管理方案
* Tailwind CSS
* RSS
* Sitemap
* Vercel

### 11.3 技术栈方案 C：极简静态生成器

适合：希望网站极快、依赖少、长期稳定。

建议：

* Hugo / Zola / Eleventy
* Markdown
* SCSS
* Netlify / Cloudflare Pages

### 第一版推荐

推荐使用 Astro + Markdown + Tailwind CSS + Vercel / Cloudflare Pages。

理由：

* 性能好
* 适合内容型个人站
* 开发成本低
* 方便写博客
* 组件化灵活
* 后续可扩展

## 13. 性能要求

### 13.1 核心指标

* 首页首屏加载尽量控制在 2 秒以内。
* Lighthouse Performance 目标：90+
* 图片使用 WebP / AVIF。
* 首屏头像和项目封面需压缩。
* 非关键动画延迟加载。
* 字体文件尽量减少。

### 13.2 技术要求

* 图片懒加载
* CSS 按需加载
* JS 控制体积
* 避免引入过重动画库
* 静态资源 CDN 加速

---

## 14. 可访问性要求

1. 文本与背景对比度达标。
2. 所有图片提供 alt 文本。
3. 链接文本应具备可理解含义。
4. 页面支持键盘导航。
5. hover 状态不能作为唯一交互提示。
6. 字号不低于 16px。
7. 动画需尊重 prefers-reduced-motion。

## 18. 验收标准

### 18.1 首页验收

* 用户进入首页后能在首屏看到姓名、定位、简介和核心链接。
* 重点项目至少展示 3 个。
* 联系方式清晰可点击。
* 移动端展示正常，无横向滚动问题。

### 18.2 项目展示验收

* 每个项目信息完整。
* 至少包含标题、简介、角色、技术栈、链接。
* 重点项目排序合理。
* 外链可正常打开。

### 18.3 时间线验收

* 时间线按时间倒序展示。
* 每条经历包含时间、标题和描述。
* 移动端阅读体验良好。

### 18.4 性能验收

* 首页 Lighthouse Performance 达到 90 分以上。
* 图片均完成压缩。
* 页面无明显卡顿。
* 动画不影响主要内容加载。

### 18.5 SEO 验收

* 首页和主要页面有独立 title 与 description。
* sitemap.xml 正常生成。
* 社交平台分享时能显示标题、描述和图片。

## 20. 风险与注意事项

1. 内容不足会导致网站显得空，需要优先整理真实项目与经历。
2. 视觉不要过度模仿参考网站，应保留个人识别度。
3. 动效不宜过多，否则会影响内容阅读。
4. 项目描述应突出结果和个人贡献，避免流水账。
5. 网站结构应方便长期维护，不应依赖复杂后台。
6. 个人联系方式需要注意隐私保护，例如微信可用二维码弹窗，不直接暴露手机号。

## 25. 总结

该个人作品集网站应以“真实、清晰、长期积累”为核心。视觉上保持极简和技术感，内容上突出个人身份、代表项目、成长时间线和联系方式。第一版应优先完成首页、项目展示、时间线和联系入口，后续再逐步扩展项目详情、博客、主题切换和多语言能力。
